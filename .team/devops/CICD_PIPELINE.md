# CI/CD Pipeline Architecture

> DevOps documentation for the Claw Agents Provisioner CI/CD pipeline.

---

## Pipeline Overview

The CI/CD pipeline is implemented via **GitHub Actions** and is triggered on:
- **Push** to `main` or `develop` branches
- **Pull requests** targeting `main`

The pipeline runs 8 parallel jobs covering linting, building, validation, and security scanning.

## Job Architecture

```
ci.yml
  |
  +-- shellcheck          Lint all .sh files (shellcheck --severity=warning)
  +-- hadolint            Lint all 5 Dockerfiles (hadolint-action)
  +-- ruff                Lint all Python files (ruff check)
  +-- docker-build        Matrix build: 4 agents x Docker (parallel)
  |     +-- zeroclaw      docker build -f zeroclaw/Dockerfile
  |     +-- nanoclaw      docker build -f nanoclaw/Dockerfile
  |     +-- picoclaw      docker build -f picoclaw/Dockerfile
  |     +-- openclaw      docker build -f openclaw/Dockerfile
  +-- compose-validate    Validate docker-compose.yml syntax
  +-- assessment          Validate example assessment JSONs
  +-- datasets            Check dataset structure and row counts
  +-- security            Scan for secrets and PII in tracked files
```

## Linting Jobs

### ShellCheck
- Scans every `.sh` file in the repository
- Severity threshold: `warning`
- Covers: `claw.sh`, `entrypoint.sh` (x4), `install-*.sh` (x4), `provision-base.sh`, `healthcheck.sh`

### Hadolint
- Lints all 5 Dockerfiles:
  - `zeroclaw/Dockerfile`
  - `nanoclaw/Dockerfile`
  - `picoclaw/Dockerfile`
  - `openclaw/Dockerfile`
  - `finetune/Dockerfile.finetune`
- Uses `hadolint-action@v3.1.0`
- Failure threshold: `warning` (non-blocking for now)

### Ruff
- Lints all Python files (`*.py`)
- Output format: `github` (inline annotations on PRs)
- Covers assessment pipeline scripts and fine-tuning scripts

## Docker Build Matrix

The matrix build runs **4 parallel jobs**, one per agent:

| Agent     | Dockerfile                   | Timeout | Notes                                |
|-----------|------------------------------|---------|--------------------------------------|
| ZeroClaw  | `zeroclaw/Dockerfile`        | 15 min  | Multi-stage: download binary         |
| NanoClaw  | `nanoclaw/Dockerfile`        | 15 min  | Clones repo, npm install             |
| PicoClaw  | `picoclaw/Dockerfile`        | 15 min  | Multi-stage: Go build                |
| OpenClaw  | `openclaw/Dockerfile`        | 15 min  | Node.js 22, pnpm install             |

Builds are set to `continue-on-error: true` because upstream repositories may be unavailable during CI. The image verification step reports warnings instead of failures.

## Validation Jobs

### Compose Validation
- Creates a dummy `.env` from `.env.template`
- Runs `docker compose config --quiet` to validate YAML syntax

### Assessment Validation
- Runs `assessment/validate.py` against any `*.example.json` files
- Skips gracefully if validator is not yet implemented

### Dataset Validation
- Counts dataset directories, metadata files, and data files
- Runs `finetune/validate_datasets.py` if available
- Reports totals for monitoring dataset completeness (target: 50/50)

## Security Scanning

Three security checks run on every CI run:

1. **No `.env` files tracked** — Fails the build if any `.env` file is committed
2. **No API key patterns** — Warns if `sk-ant-`, `sk-or-`, `xoxb-`, `hf_` patterns found in tracked files (excludes `.env.template` and docs)
3. **No client assessment files** — Fails if any `client-assessment*.json` (non-example) is tracked

## Local CI Testing

Use [act](https://github.com/nektos/act) to run the CI pipeline locally:

```bash
# Run all jobs
act push

# Run a specific job
act push -j shellcheck
act push -j docker-build

# Run with matrix filter
act push -j docker-build --matrix agent:zeroclaw
```

## Branch Protection Rules (Recommended)

Configure on GitHub:
- Require PR reviews before merging to `main`
- Require status checks to pass: `shellcheck`, `security`, `compose-validate`
- Require branches to be up to date before merging
